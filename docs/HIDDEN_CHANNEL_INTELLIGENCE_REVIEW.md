# Hidden Channel Intelligence Implementation Review

Date: 2026-05-04

This document records the implementation review findings for `.sisyphus/plans/hidden-channel-intelligence.md` and the fixes required before the plan can be considered complete.

## Findings

- **Critical: Telegram first-run crawl window is wrong.** `TelegramIntelligenceCrawler` used Telethon `offset_date` as if it meant "since this time". Telethon iterates newest-to-oldest by default, so this can fetch the wrong side of the 24-hour cutoff.
- **High: Postgres migration and TTL cleanup disagree.** `003_intelligence_schema.sql` created `raw_text TEXT NOT NULL`, while raw TTL cleanup nulls `raw_text` after expiry.
- **High: Production migration path skips intelligence schema.** `docker-entrypoint.sh migrate-postgres` applied only `001_init.sql`.
- **High: Telethon runtime dependency missing.** The Telegram intelligence collector imports Telethon dynamically, but the project dependency list did not install it.
- **High: Related candidates were not generated by ingestion.** The merge engine exposed `create_related_candidates`, but the ingestion pipeline never called it after semantic indexing.
- **High: Primary label mismatch.** The plan requires label `crypto`; the implementation used uppercase `CRYPTO`.
- **Medium: V2EX rate-limit and v2 pagination behavior was incomplete.** The crawler recorded rate-limit headers but continued crawling at quota exhaustion, and v2 requests sent unsupported pagination parameters.
- **Medium: Intelligence semantic API ignored hourly windows.** `/intelligence/search` only parsed day windows such as `7d`, while the plan also requires hour windows such as `24h`.
- **Medium: Telegram command history retained raw evidence.** `/intel_detail ... raw` logged full response text, including raw evidence, outside raw TTL cleanup.
- **Medium: Telegram datasource listing could expose REST query secrets.** It displayed REST endpoints verbatim instead of stripping query strings/fragments.
- **Medium: Raw dedupe collapsed repeated identical evidence.** Dedupe used `(source_type, source_id, content_hash)`, preventing repeated identical messages with different external IDs from increasing evidence counts.
- **Process gap: Required evidence artifacts were missing.** The plan marks tasks complete without the required `.sisyphus/evidence/task-*` files.

## Verification Notes

- Focused intelligence tests passed after installing `uv` in the review environment.
- Real Postgres/pgvector tests still require `TEST_DATABASE_URL` and were skipped without it.
- `mypy` currently fails repository-wide, including some intelligence files; this remains a separate hardening requirement unless fixed as part of this remediation.
