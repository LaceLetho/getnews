Implementation
--------------
Extended crypto_news_analyzer/intelligence/topic_prompts.py with TopicPromptWorkflowService:

- create_draft_topic(theme) -> TopicPrompt
  Creates an IntelligenceTopic in draft lifecycle status, generates a prompt draft
  via TopicPromptGenerator + LLM, persists both to repository.

- revise_prompt(topic_id, feedback) -> TopicPrompt
  Fetches the most recent prompt for the topic, uses TopicPromptReviser + LLM to
  create a new version with incremented version number, persists as draft.

- replace_prompt_manual(topic_id, prompt_text) -> TopicPrompt
  Creates a new draft version with manually-supplied text. Rejects empty text and
  text exceeding max_prompt_length (default 50000 chars). Preserves prior audit_history.

- confirm_prompt(topic_id, prompt_version_id) -> TopicPrompt
  Validates the prompt is a draft and belongs to the topic. Archives any currently
  active prompt for the topic, then activates the confirmed draft.

- edit_active_prompt(topic_id, new_prompt_text) -> TopicPrompt
  Fetches the active prompt, creates a new draft version with incremented version
  and the new text. Original findings keep their original prompt_version_id.
